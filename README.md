# SmartCook 食光再造 -  智慧永續餐食決策系統 

本研究提出SmartCook（食光再造）作為一套整合式的智慧餐食輔助系統，結合YOLO 深度學習食材辨識、個人化飲食偏好與過敏原過濾、自建營養分析API、以及LINE
Bot 零門檻互動介面，協助使用者以更直覺、更快速、更科學的方式管理剩餘食材並制定合適的飲食方案。

## 系統核心創意與特色
* **AI 食材辨識**：導入 YOLOv8 深度學習影像辨識技術，支援多物體與小型物件（如蒜頭、辣椒）同時辨識，取代傳統繁瑣的文字輸入。
* **智慧三重過濾機制**：結合會員資料庫，在食譜比對時自動排除使用者的過敏原（如花生、牛奶），並符合個人飲食偏好（如素食、低卡）。
* **自建營養分析 API**：串接政府公開資料，自動計算料理的熱量、蛋白質、脂肪與碳水化合物，落實健康飲食管理。
* **低門檻互動介面**：以前端 LINE Bot 作為直覺的操作入口，利用 Flex Message Carousel 呈現美觀的卡片式 UI 成果。
  
## 系統架構
* **使用者輸入**：透過Line聊天界面拍照上傳食材、輸入文字或設定偏好。
* **後端串接**:接收Line Webhook事件，進行分流、事件分類及邏輯處理。
* **圖片辨識**:接收到照片會觸發YOLOv8模型進行多物件食材辨識。
* **資料對比**:根據辨識結果從MySQL食譜庫進行交集比對，並套用會員飲食偏好、排除過敏原、份量需求進行篩選。
* **輸出結果**：將最終篩選出的食譜和營養資訊，包裝成 Carousel做回傳。
  
## 開發工具與技術
* **後端主體語言**：Python 
* **Web 伺服器框架**：Flask (處理 LINE Webhook 事件、狀態管理、API 整合)
* **影像辨識模型**：YOLOv8 (基於 Kaggle 蔬果資料集與自建影像資料集進行訓練)
* **資料庫管理**：MySQL (儲存會員偏好設定、過敏原與結構化食譜)
* **資料擷取**：Python 爬蟲 (Requests, BeautifulSoup, Selenium) 自動收集與清洗 iCook 熱門食譜資料
* **第三方 API**：LINE Messaging API, 衛福部食品營養成分資料庫
